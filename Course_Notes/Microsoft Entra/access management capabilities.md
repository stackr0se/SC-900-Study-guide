# Conditional Access
At their simplest, Conditional Access (CA) policies are if-then statements.

Assignments determine which signals to use
- Users, groups, workload identties, directory roles.
- Cloud apps or actions
- Sign-in and user risk detection
- Device or device platform
- IP location and more...

Access controls determine how a policy is enforced
- Block access
- Grants access-Require one or more conditions to be met before granting access
- Session control-Enable a limited experience

# Microsoft Entra Global Secure Access

GSA converges Zero Trust network, identity, and endpoint access controls to secure access to any ap or resource, from any location, device, or identity.
- Microsoft Entra Iinternet Access-secures access to SaaS applications, including Microsoft Services, and public internet apps.
- Microsoft Entra Private Acess-provides your users secure access to your private, corporate resources.

# Microsoft Entra roles and role-based access control (RBAC)

Microsoft Entra ID roles control permissions to manage Microsoft Entra resources.
- Bult-in roles
- Custom roles
- Categories of Microsoft Entra roles are:
   - Microsoft Entra specific
   - Service specific
   - Cross service
- Only grant the access users need
