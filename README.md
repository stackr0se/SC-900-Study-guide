## 📋 Official Exam Skills Checklist

### 📂 Domain 1: Concepts of Security, Compliance, and Identity (10–15%)
- [ ] **Describe security and compliance concepts and methodologies**
  - [ ] Describe the Zero Trust methodology (Verify explicitly, Use least privileged access, Assume breach)
  - [ ] Describe the shared responsibility model
  - [ ] Define defense-in-depth
  - [ ] Describe common security threats (Malware, Phishing, Ransomware, DDoS)
  - [ ] Describe encryption (Encryption at rest, Encryption in transit, Hashing)
- [ ] **Define identity concepts**
  - [ ] Define identity as the primary security perimeter
  - [ ] Define authentication (AuthN) and authorization (AuthZ)
  - [ ] Describe what identity providers are
  - [ ] Describe the concept of Federated services

### 📂 Domain 2: Capabilities of Microsoft Entra (25–30%)
- [ ] **Describe services and identity types in Microsoft Entra**
  - [ ] Describe the Microsoft Entra family (Entra ID, Governance, Protection)
  - [ ] Describe identity types (Users, Devices, Groups, Service Principals/Applications)
  - [ ] Describe the concept of hybrid identity (Microsoft Entra Connect)
  - [ ] Describe external identity types (Guest users, B2B, B2C)
- [ ] **Describe authentication capabilities of Microsoft Entra**
  - [ ] Describe authentication methods (MFA, Windows Hello for Business, FIDO2)
  - [ ] Describe Self-Service Password Reset (SSPR)
  - [ ] Describe password protection and management capabilities (Smart Lockout, Custom Banned Lists)
- [ ] **Describe access management capabilities of Microsoft Entra**
  - [ ] Describe what Conditional Access is, its signals, and its benefits
  - [ ] Describe Microsoft Entra ID roles and Role-Based Access Control (RBAC)
- [ ] **Describe identity protection and governance capabilities**
  - [ ] Describe Privileged Identity Management (PIM) for just-in-time access
  - [ ] Describe Microsoft Entra Identity Governance (Access Reviews, Entitlement Management)
  - [ ] Describe Microsoft Entra ID Protection (Risk-based Conditional Access)

### 📂 Domain 3: Capabilities of Microsoft Security Solutions (35–40%)
- [ ] **Describe core security capabilities in Azure**
  - [ ] Describe Network Security Groups (NSGs) and Application Security Groups (ASGs)
  - [ ] Describe Azure Firewall, Azure DDoS Protection, and Web Application Firewall (WAF)
  - [ ] Describe Azure Bastion
  - [ ] Describe Azure Key Vault
- [ ] **Describe security management capabilities of Azure**
  - [ ] Describe Microsoft Defender for Cloud
  - [ ] Describe Cloud Security Posture Management (CSPM) and the Microsoft Secure Score
  - [ ] Describe Cloud Workload Protection Platform (CWPP) and Defender plans
  - [ ] Describe security baselines for Azure
- [ ] **Describe threat protection capabilities of Microsoft Defender XDR**
  - [ ] Describe Microsoft Defender XDR services (Endpoint, Office 365, Identity, Cloud Apps)
  - [ ] Describe the Microsoft Defender portal and incident management
- [ ] **Describe security capabilities of Microsoft Sentinel**
  - [ ] Define Security Information and Event Management (SIEM) and Security Orchestration, Automation, and Response (SOAR)
  - [ ] Describe the role and data connectors of Microsoft Sentinel

### 📂 Domain 4: Capabilities of Microsoft Compliance Solutions (20–25%)
- [ ] **Describe Microsoft service trust, privacy, and regulatory principles**
  - [ ] Describe Microsoft’s Privacy Principles and privacy management
  - [ ] Describe the offerings of the Service Trust Portal
  - [ ] Describe Microsoft Priva capabilities
- [ ] **Describe compliance management capabilities of Microsoft Purview**
  - [ ] Describe the Microsoft Purview compliance portal and Compliance Manager
  - [ ] Describe the benefit of compliance scores
- [ ] **Describe information protection and data lifecycle management in Purview**
  - [ ] Describe data classification capabilities (Sensitivity labels, Sensitive Information Types)
  - [ ] Describe Data Loss Prevention (DLP)
  - [ ] Describe Retention Policies and Retention Labels
- [ ] **Describe risk, governance, and discovery capabilities in Purview**
  - [ ] Describe Insider Risk Management
  - [ ] Describe eDiscovery capabilities (Standard vs. Premium)
  - [ ] Describe Audit capabilities
